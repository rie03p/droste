# Droste（再帰ズーム）の原理

「Droste 効果」とは、ある画像の中に同じ画像の縮小版が含まれ、その中にもまた縮小版が…と無限に入れ子になっている状態を指す（ココアの缶の絵が有名）。
このエフェクトの目的は、**通常の画像を「拡大していくと同じ画像に戻る」自己相似な絵に変え、それをシームレスにループさせる**こと。渦（ひねり）は加えない純粋な再帰ズームを扱う。

## 1. 核心は「スケール不変性」

鍵になるのは、ある一点 `p*`（**蓄積点**）を中心に **f 倍に拡大しても見た目が変わらない画像** を作ること。

```
B(p*まわりで f 倍に拡大した点) = B(元の点)
```

このような画像 `B` を「f 倍スケールに対して不変」「自己相似」と呼ぶ。
もし画像がこの性質を持てば、蓄積点へ向かってカメラをズームしていったとき、ちょうど f 倍ズームしたところで画面は元と完全に一致する。これが「拡大すると同じ画像に戻る」の正体であり、ループの土台になる。

## 2. 窓（収縮写像）で自己相似性を作る

任意の写真は当然そのままでは自己相似ではない。そこで、画像の上に**窓**（小さな矩形）を 1 つ決め、「窓の中には画像全体の縮小コピーが入っている」と取り決める。

窓の中心を `c`、窓の大きさ（画面に対する割合）を `size` とすると、「画面全体を窓へ縮める」写像は

```
T(p) = (p - 0.5)·size + c
```

これは **一点 `p*` を中心に size 倍へ縮小する相似変換**になっている。その動かない点（不動点）が蓄積点で、

```
p* = (c − 0.5·size) / (1 − size)
```

窓の中にはコピーが入り、そのコピーの中の窓にもまたコピーが…と `T` を繰り返し適用した入れ子が無限に続く。各段は前の段を `size` 倍したものなので、全体は「`p*` まわりの 1/size 倍スケールに対して不変」＝自己相似になる。つまり **f = 1/size** が 1 段あたりの拡大率である。

> 窓の縁は「額縁」として残る。これは picture-in-picture 型 Droste の本質的な特徴で、論文（de Smit–Lenstra）の Escher 版では log-polar の等角写像がこの縁を渦の中に溶かして見えなくしている。本エフェクトは縁を残す素朴版にあたる。

## 3. 「焼き込む」のではなく「展開して取り直す」

入れ子コピーを画像として事前に描き込む（焼き込む）方法もあるが、それだと内側のコピーほど解像度が落ち、深くズームするほどボケる。すると 1 ループの始点と終点で鮮明さが食い違い、継ぎ目が見える。

代わりに本実装は、**表示すべき点が窓の中に入っていたら、写像 `T` を逆にたどって（`T⁻¹` で拡大して）窓の外に出るまで戻し、最後に必ず元画像（レベル 0）を参照する**という考え方を採る。

```
見たい点 q が窓の中にある間：
    q ← T⁻¹(q)          （窓 → 画面全体 へ引き伸ばす）
窓の外に出たら：
    元画像を q で参照する
```

何段ネストしていても、最終的に色を取り出すのは毎回「元画像そのもの」。だから**どのズーム段でも鮮明度が一定**になる。理想的には無限解像度の自己相似画像を、実際には有限の元画像を再帰展開で代用している、という関係。

## 4. 連続ズームとログスケールの周期性

カメラのズーム量を `s`（`s=1` が等倍、小さいほど寄る）とすると、見たい点は

```
q = p* + (vUv − p*)·s
```

`s` を `1` から `size` まで動かすと、ちょうど 1 段ぶん潜る。

ここで効いてくるのが**スケールを対数で見ると周期的**だという事実。スケール `s` の対数 `log s` を `0` から `−log f`（= `log size`）へ動かすのが 1 周期で、

- `s = 1`（log s = 0）のフレーム … 元画像そのもの
- `s = size`（log s = −log f）のフレーム … `q = T(vUv)` となり、これは必ず窓の中。1 回展開すると `vUv` に戻るので、結局 `s=1` と**同じ点・同じ解像度**を参照する

したがって両端のフレームは完全に一致し、`s` が `size` を過ぎて `1` に戻る瞬間に継ぎ目が出ない＝シームレスループになる。
アニメーションは内部的に `s = exp(−mod(offset, log f))` として `offset` を線形に流すだけでよい。GIF 書き出しも同じ掃引なのでそのままループ素材になる。

> ここで `s` の最小値（`1/f`）が窓サイズ `size` に**厳密に一致**していることが不可欠。`f` と `1/size` がずれると `s` が窓サイズに届かず、ループの折り返し点に達する前にジャンプしてしまう。常に `f = 1/size` とする。

## 5. なぜこれが論文の「単純な場合」なのか

de Smit–Lenstra の "The Mathematical Structure of Escher's Print Gallery" は、複素平面上で `z ↦ z^α`（α は複素数）という等角写像により、自己相似なスケール方向を**斜めにひねって**渦（Print Gallery 効果）を作る。

本エフェクトはそのひねりを 0 にした特別な場合に相当する。ひねりがないので角度方向の巻き込みが起きず、放射状の歪みのない**まっすぐな無限ズーム**になる。ひねりを足したものが本アプリの「Escher 渦」エフェクトで、そちらは log-polar 座標での回転として実装している。

## まとめ

1. 一点 `p*` まわりの f 倍スケールに不変な画像を作れば「拡大で同じ画像」になる。
2. 窓（収縮写像 `T`）に画像自身を入れることで、任意画像を自己相似化する。`f = 1/size`、`p*` は `T` の不動点。
3. 入れ子は焼き込まず、`T⁻¹` で展開して毎回元画像を参照することで鮮明度を一定に保つ。
4. スケールの対数が周期的なので、`s` を `1↔size` で往復させればシームレスにループする。
