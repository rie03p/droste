# Droste（再帰ズーム）の実装

このドキュメントは「Droste（再帰ズーム）」エフェクトの仕組みだけを説明する。
目的は **通常の画像を「拡大していくと同じ画像に戻る」無限ズームに変換し、シームレスにループさせる** こと。
渦（ツイスト）は付けない純粋な再帰ズームで、Escher 渦は別のエフェクト（log-polar 写像）。

## 全体像

1. ユーザーが画像の上に **窓（矩形）** を 1 つ指定する。窓はビューと同じ縦横比。
2. その窓に「画像全体の縮小コピー」が入る、と解釈する。窓に入った先にもまた窓があり…と無限に入れ子になる。
3. 窓の中心へ向かってズームしていくと、ちょうど 1 段ぶん（窓のサイズ分）拡大したところで元の画像と完全に一致する。これがシームレスループになる。

ポイントは **入れ子画像を事前に焼き込まず、フラグメントシェーダの中で座標を再帰展開して毎回フル解像度の元画像を参照する** こと。焼き込み方式だと内側のコピーが低解像度になり、ズーム段ごとに鮮明度が変わってループの継ぎ目が見えてしまう。

## 数式

窓の中心を `c = (cx, cy)`、窓の大きさ（ビューに対する割合）を `size` とする。`size = 1/f`（f は 1 段あたりの拡大率）。

ビュー全体 `[0,1]²` を窓へ移す写像 `T` を次で定義する（座標は uv 空間 `[0,1]²`）：

```
T(p) = (p - 0.5) * size + c
```

`T` は「ある点 p* を中心に size 倍へ縮小する相似変換」になっている。その不動点（＝ズームの蓄積点）p* は

```
p* = (c - 0.5 * size) / (1 - size)
```

窓に画像全体が入る、という条件は

```
B(T(p)) = B(p)        … 画像 B は p* まわりの size 倍スケールに対して不変
```

と書ける（B は表示される無限入れ子画像）。これは「p* を中心に f=1/size 倍に拡大しても同じ画像」という自己相似性そのもの。

## シェーダ（中核）

`src/effects.ts` の `DROSTE_PLAIN`。出力ピクセルごとに次を行う：

```glsl
vec2  winC  = vec2(cx, cy);
vec2  pstar = (winC - 0.5 * size) / (1.0 - size);   // 蓄積点

float lnf = log(f);
float s   = exp(-mod(u_offset, lnf));               // s ∈ (1/f, 1] = (size, 1]
vec2  q   = pstar + (vUv - pstar) * s;              // 蓄積点へズームイン

// q が窓に入っていれば T^-1 で展開し、窓から出るまで繰り返す
for (int i = 0; i < 96; i++) {
  if (abs(q.x - cx) < 0.5 * size && abs(q.y - cy) < 0.5 * size) {
    q = (q - winC) / size + 0.5;                    // = T^-1(q)
  } else break;
}
outColor = texture(level0Image, q);                 // 常にレベル0をフル解像度で参照
```

- `level0Image` は入れ子を焼いていない **元画像そのもの**（`makeCover` でビュー比に整えただけ）。
- 窓に入った座標を `T^-1` で「展開」して、窓の外（＝元画像の地の部分）に出るまで戻す。何段ネストしても最後に参照するのは元画像なので、どのズーム段でも鮮明度が一定。
- ループ上限 96 は、窓が大きく（f が小さく）展開が遅いときに蓄積点付近まで解像するための余裕。窓が小さいときは数回で `break` する。

## なぜシームレスにループするか

アニメーションは `u_offset` を `0 → lnf` で動かす（`src/components/ShaderCanvas.tsx` が毎フレーム加算し、周期 `lnf = log(f)` で `wrap`）。これにより `s = exp(-mod(offset, lnf))` は `1 → size` を一周する。

- `s = 1` のフレーム：`q = vUv`。窓の外なら元画像 `B(vUv)` をそのまま参照。
- `s = size` のフレーム：`q = pstar + (vUv - pstar) * size = T(vUv)`。`T(vUv)` は必ず窓の中なので 1 回展開され、`(T(vUv) - c)/size + 0.5 = vUv` に戻る。あとは `s=1` と同じ経路をたどる。

つまり **`s=size` のフレームは `s=1` のフレームと同一ピクセル・同一解像度** になる。`offset` が `lnf` に達して `0` に戻る瞬間（`s: size → 1`）に継ぎ目が出ない。GIF 書き出し（`src/gif/exportGif.ts`）も同じ `offset` 掃引なのでシームレス。

> 重要：`s` の最小値（`1/f`）が窓サイズ `size` に**正確に**一致している必要がある。`f` を `1/size` と別の値にクランプすると、`s` が窓サイズに届かずループの継ぎ目に達する前に折り返してしまう（過去の不具合）。`f = 1/size` を厳守する。

## アスペクト比と座標の注意点

- **テクスチャはビューと同じ縦横比で焼く**（`makeCover(square, view.width, view.height)`）。サンプリングは常に `vUv ∈ [0,1]²` なのでテクスチャ外を参照せず、16:9 などでも端に同じ画像が繰り返す問題が出ない。
- **Y 軸の向き**：UI/2D キャンバスは上→下、GL の `vUv.y` は下→上。そのためシェーダへ渡す窓の y は `1 - cy` に反転している（`App.tsx`）。窓の x はそのまま。

## パラメータ早見

- `size`（窓の大きさ, 0.002〜0.9）：小さいほど 1 段の拡大率 `f = 1/size` が大きい＝1ループで深く潜る。
- `cx, cy`（窓の中心, 0〜1）：ズームしていく先。窓に画像自身が埋め込まれるので、模様が連続する場所に置くと額縁（窓の縁）が目立ちにくい。
